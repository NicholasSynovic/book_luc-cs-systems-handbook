##########
 Overview
##########

Prim is the Loyola University Chicago (LUC) Computer Science (CS)
Department's primary *testing* cluster computer. It is intended to
provide Dijkstra system administrators with a platform to test new
software, node architectures, and other features in a low risk, isolated
environment. These features, once tested and configured can then be
implemented in Dijkstra with (ideally) limitted downtime.

Prim (like Dijkstra) is a *heterogenous* system; Prim nodes are not
necessarily the same hardware, system architecture, or even physical.
Prim is designed as a specification that allows any platform to spin up
a compatible system. This allows for the testing of new hardware
architectures or software systems without being constrained to a
particular hardware platform.

As we expect Dijkstra to grow and expand to the needs of our users, Prim
is a living specification. When new hardware architectures are
introduced or system features are requested, Prim will be adapted and
updated to test said features prior to deploying on Dijkstra.

When implementing Prim, it is recommended to leverage a hardware
implementation over a software emulation. As the current software
implementations are dependent upon potentially different hardware
platforms, there is no gaurantee that Prim will operate equivacaly on a
new hardware platform.
